Модуль 0: Подготовка среды и Git

ТЕОРИЯ:
- Git - система контроля версий для отслеживания изменений в файлах
- GitHub - платформа для хостинга Git-репозиториев
- Docker - платформа для контейнеризации приложений
- PySpark - Python API для Apache Spark

КЛЮЧЕВЫЕ КОМАНДЫ:
- git config --global user.name "name" - настройка пользователя
- ssh-keygen - генерация SSH-ключа
- git clone - клонирование репозитория
- git add . - добавление файлов к коммиту
- git commit -m "message" - создание коммита
- git push - отправка изменений на GitHub
- docker-compose up -d - запуск контейнеров
- docker logs container_name - просмотр логов

ПРАКТИКА:
- Настроен Git с SSH-ключом
- Создан репозиторий на GitHub
- Установлена структура проекта
- Запущен Docker-контейнер с PySpark
- Созданы генераторы тестовых данных
- Проверена работоспособность PySpark

BEST PRACTICES:
- Использование SSH-ключей для GitHub (безопаснее, чем пароли)
- .gitignore для исключения ненужных файлов из репозитория
- Организованная структура проекта
- Связывание локальных изменений с удаленным репозиторием
